# Structure-Preserving Monte Carlo for Trading-Strategy Timing — Code & Data Companion

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.20695539.svg)](https://doi.org/10.5281/zenodo.20695539)

Reproducibility companion for the paper *"Structure-Preserving Monte Carlo Inference for
Trading-Strategy Timing"* (gold futures, GC=F, 2002-03-04 to 2026-04-02).

The framework tests whether a strategy's **entry placement** carries information beyond its
structural exposure, by holding fixed the realized price path, trade count, holding-period
multiset, inter-trade-gap multiset, capital weights, and direction sequence, and randomizing
**only** the placement of trades on the calendar.

Every table and figure in the paper is reproducible from the scripts and data here. The headline
result (no robust, measure-invariant entry-placement skill on GC=F) and the two confirmatory
analyses (a real-data positive control; a 322-test cross-asset multiplicity panel) all reproduce
with the commands below.

## Repository layout


## Setup

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
export SAVE_OUTPUTS=1            # persist regenerated figures/CSVs
export MONTE_CARLO_SEED=42       # the seed used throughout the paper (default)
```

Python 3.11+ recommended. Runs are deterministic given `MONTE_CARLO_SEED=42`.

## Reproducing each result

All commands are run from the repository root. Outputs are written under `Data_Clean/` and `Charts/`.

| Paper element | Command |
|---|---|
| **Table: Synthetic validation** + calibration & power figures | `SYNTHETIC_REPLICATIONS=1500 SYNTHETIC_POWER_REPLICATIONS=400 SYNTHETIC_SIMULATIONS=2000 python Code/synthetic_timing_experiments.py` |
| **Table: Positive control** + skill-curve & histogram figures | `python Code/positive_control_power.py` |
| **Table: Cross-asset multiplicity** + BH figure | `python Code/cross_asset_multiplicity.py` |
| **Table: Schedule-measure sensitivity** (baseline column reproduces the central results table exactly) | see the three measure commands below |
| **Figure: Multi-asset walk-forward inference** | `python Code/regen_multi_asset_inference.py` |

Schedule-measure sensitivity (three measures, $M=5{,}000$, seed-42 spawn protocol):

```bash
MEASURE_LABEL=baseline \
  python Code/schedule_measure_sensitivity.py
MONTE_CARLO_MIN_LEADING_INDEX=200 MEASURE_LABEL=leading_gap_200 \
  python Code/schedule_measure_sensitivity.py
MONTE_CARLO_CONTEXT_MATCHING=1 MEASURE_LABEL=context_matched \
  python Code/schedule_measure_sensitivity.py
```

The `baseline` column reproduces the paper's central results table p-values exactly
(e.g. Volatility Squeeze Breakout `p = 0.0534`, minimum panel `p = 0.0534`, zero rejections at 5%),
confirming the analysis harness matches the canonical pipeline.

## Key modules

- `monte_carlo.py` — the structure-preserving timing null (gap-permutation measure $\mathbb{Q}$),
  RCSI / RCSI$_z$, one-sided Monte Carlo p-values, Benjamini–Hochberg adjustment, and the
  context-matched / leading-gap measure variants (selected by environment).
- `synthetic_timing_experiments.py` — five controlled worlds for Type-I calibration and power,
  with a self-contained Kolmogorov–Smirnov uniformity test (no SciPy dependency).
- `positive_control_power.py` — injects a tunable fraction of genuinely well-timed entries on the
  real GC=F path and scores them with the same null (the real-data power / positive control).
- `cross_asset_multiplicity.py` — pools every completed structure-preserving test on disk
  (322 strategy-by-asset tests across 47 instruments) and applies BH and Bonferroni control.
- `schedule_measure_sensitivity.py` — re-evaluates the GC=F panel under alternative schedule
  measures without re-running any backtest.

## Data provenance and licensing

Price data (GC=F and the cross-asset universe) is sourced from **Yahoo Finance** via the
`yfinance` package; `Code/data_loader.py` performs the download (`period="max"`). This repository
ships the **derived** artifacts needed to reproduce the paper — regime-labeled OHLCV
(`*_regimes.csv`), realized trade logs (`GC=F_*_trades.csv`), and per-asset result tables
(`*_full_comparison.csv`) — rather than redistributing raw vendor feeds. Yahoo Finance data is
subject to Yahoo's terms of use; users who wish to refetch raw bars should do so themselves via
`Code/data_loader.py`. Reported values may differ by fractions of a basis point across data
vintages; the included artifacts are the exact ones underlying the manuscript.

## Determinism and faithfulness

Every number in the manuscript traces to a saved artifact in `Data_Clean/` or to a figure in
`ProvidedFigures/`. The Monte Carlo runs are seeded (`MONTE_CARLO_SEED=42`); rerunning the
commands above reproduces the reported tables and figures.
